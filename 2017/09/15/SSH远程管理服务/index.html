SSH 简介

SSH（安全外壳协议）

  1. 为Secure Shell的缩写，SSH为建立在应用层和传输层基础上的安全协议。
  2. telnet这以远程管理协议为明文传输，危害极大。
  3. tcpdump -i eth0 -nnX port 21 来抓取流量
  4. ps aux | grep sshd 查看ssh进程
  5. netstat -tlun | grep 22 查看22端口

SSH端口

  1. SSH端口：22
  2. Linux中守护进程：sshd
  3. 安装服务：OpenSSH
  4. 服务端主程序：/usr/sbin/sshd
  5. 客户端主程序：/usr/bin/ssh

相关配置文件

  1. 服务端配置文件：/etc/ssh/sshd_config
  2. 客户端配置文件：/etc/ssh/ssh_config

SSH加密原理

  1. 文件的属性->高级->加密内容以保护数据（只有管理员能打开）
  2. 对称加密算法:加密和解密使用的是同一个密钥，一旦泄露，可能造成撞库攻击
  3. 非对称加密算法：分为公私钥
  4. SSH加密的是网络传输中的数据，本地键盘被记录的情况下被窃取的是没有被加密的数据
  5. 只有第一次访问SSH需要公钥。

  6. 通过配置文件修改IP

    1. 重启network
